# 4つのゼロコスト・エンジン（具体スタック）

各エンジンは同じ背骨の「設定違い」。データ源と出力を差し替えるだけ。

---

## ① 自律リサーチエンジン
**「毎日勝手に調べて蓄積し続ける」情報優位の自動化。**（Grok級ループの汎用版）

```
GAS タイマー(毎朝6:00)
  → UrlFetchApp で無料API/RSS を取得
  → Gemini API(無料) で要約・重要度スコアリング
  → Sheets に追記（履歴が育つ）
  → 重要度高だけ Gmail 通知
  → Claude Code が Sheets を読んで深掘り分析・レポート執筆
```

**適用例**：業界ニュース監視 / 競合動向 / 市場情報集約 / 学術論文ウォッチ /
（投資）ウォッチリストの値動き＋ニュースの日次サマリー ※助言ではなく情報集約

**コスト**：ゼロ（GAS無料 + Gemini無料枠 + データAPI無料枠）

---

## ② ゼロコスト・バックエンド / マイクロSaaS
**サーバー契約なしで本格Webサービス。**

```
選択肢A: Supabase無料(500MB PG + 認証 + storage + edge functions)
選択肢B: GAS Web App(doGet/doPost) + Sheets をDB代わり
  + GitHub Pages（フロント・無料ホスティング）
  + Claude Code が全部構築
```

**適用例**：会員制ツール / フォーム集計サービス / API提供 / 予約システム

**収益化**：ユーザーの「収益化が手前」ボトルネックに直結。受託や有料ツール化へ。

---

## ③ ナレッジ / コンテンツOps
**版管理付き・多言語対応の制作配布パイプライン。**

```
Notion MCP（原稿・知識ベース）
  + Drive MCP（アセット）
  + GitHub MCP（版管理）
  + Claude Code（執筆・翻訳・編集）
  → 多言語展開 → 各チャネルへ配布
```

**適用例**：ブログ/技術記事の量産 / 宣教コンテンツの多言語展開 / ドキュメント整備

---

## ④ 意思決定ダッシュボード
**数字を毎朝、勝手に届ける。**

```
無料データAPI → Sheets（蓄積・関数で集計）
  → グラフ可視化
  → GAS で定期メールダイジェスト（要点＋グラフ画像）
```

**適用例**：事業KPI監視 / 市場ウォッチ / 在庫・売上トラッキング

---

## エンジンの組み合わせ
①で集めた情報を③で記事化し②で配布、④で効果測定——のように連鎖させると
「情報収集→制作→配布→測定」の全自動ループになる。これが最終形。
